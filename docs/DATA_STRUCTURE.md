# მონაცემთა ფოლდერი

ეს ფოლდერი შეიცავს JSON ფაილებს აპლიკაციის სტატიკური მონაცემებით.

## ფაილები

### zodiac-signs.json
შეიცავს ყველა 12 ზოდიაქოს ნიშნის სრულ ინფორმაციას:
- საბაზო ინფორმაცია (სახელი, სიმბოლო, თარიღები)
- ელემენტი და პლანეტა
- დადებითი და უარყოფითი თვისებები
- დეტალური დახასიათება
- თავსებადობა სხვა ნიშნებთან
- კარიერის მიმართულებები
- საბედნიერო რიცხვები, ფერები და დღე

### daily-horoscopes-YYYY-MM.json
თითოეული თვისთვის ცალკე ფაილი დღიური ჰოროსკოპებით.

**სახელწოდება:** `daily-horoscopes-{წელი}-{თვე}.json`  
**მაგალითი:** `daily-horoscopes-2026-02.json` (თებერვალი 2026)

**სტრუქტურა:**
```json
{
  "year": 2026,
  "month": 2,
  "horoscopes": {
    "aries": {
      "1": {
        "general": "ზოგადი პროგნოზი დღისთვის",
        "love": "სიყვარულის პროგნოზი",
        "career": "კარიერის პროგნოზი",
        "health": "ჯანმრთელობის პროგნოზი",
        "finance": "ფინანსების პროგნოზი",
        "luckyNumber": 7,
        "luckyColor": "წითელი"
      },
      "2": { ... }
    },
    "taurus": { ... }
  }
}
```

**მნიშვნელოვანი:**
- თითოეულ თვეს თავისი ფაილი სჭირდება
- დღეები იწყება 1-დან და მიდის თვის ბოლომდე
- სერვისი ავტომატურად ტვირთავს მიმდინარე თვის ფაილს
- თუ ფაილი ვერ მოიძებნა, მომხმარებელს უჩნდება შეტყობინება

## გამოყენება

### zodiac-signs.json
```typescript
import zodiacData from '../../../assets/data/zodiac-signs.json';
const signs: ZodiacSign[] = zodiacData as ZodiacSign[];
```

### daily-horoscopes
```typescript
// HoroscopeService ავტომატურად ტვირთავს მიმდინარე თვის ფაილს
const horoscope = horoscopeService.getTodayHoroscope('aries');
```

## ახალი თვის დამატება

1. შექმენით ახალი ფაილი: `daily-horoscopes-YYYY-MM.json`
2. დააკოპირეთ სტრუქტურა არსებული ფაილიდან
3. შეავსეთ ყველა 12 ნიშნისთვის ყველა დღისთვის
4. დარწმუნდით რომ year და month ველები სწორია
5. HoroscopeService ავტომატურად იტვირთება ახალი თვის დაწყებისას

## სტრუქტურის დამატება

ამ ფოლდერში შეგიძლიათ დაამატოთ სხვა JSON ფაილები:
- `planets.json` - პლანეტების ინფორმაცია
- `moon-phases.json` - მთვარის ფაზების მონაცემები
- `numerology.json` - ნუმეროლოგიის მონაცემები
- `monthly-horoscopes-YYYY-MM.json` - თვიური ჰოროსკოპები
- `yearly-horoscopes-YYYY.json` - წლიური ჰოროსკოპები
- და სხვა...
